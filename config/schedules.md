# Ad Optimizer - 스케줄 정의

이 파일은 스케줄러 설정을 사람이 읽을 수 있는 형태로 문서화합니다.
실제 스케줄 실행은 `scheduler/runner.py`가 담당합니다.

## 정기 스케줄

| Task | Cron 표현식 | 주기 | 설명 |
|------|------------|------|------|
| `collect_performance` | `0 * * * *` | 매 1시간 | 모든 플랫폼에서 캠페인 성과 데이터 수집 → DB 저장 |
| `check_market_events` | `0 */6 * * *` | 매 6시간 | 크립토 가격/뉴스 모니터링, 이상 이벤트 감지 |
| `run_agent_optimization` | `0 9 * * *` | 매일 09:00 | Claude 에이전트 → 예산/입찰가/소재 최적화 결정 |
| `generate_report` | `0 20 * * *` | 매일 20:00 | 일간 성과 리포트 생성 |

## 이벤트 드리븐 트리거 (자동)

`check_market_events` 실행 중 아래 조건이 충족되면 에이전트를 즉시 호출합니다:

| 조건 | 임계값 | 에이전트 액션 |
|------|--------|--------------|
| 비트코인 24h 가격 변동 | ≥ ±10% | 소재 전략 전환, 예산 재배분 |
| 크립토 해킹/보안 뉴스 감지 | - | 예산 상향 + "자산 보호" 소재 우선 집행 |
| 비트코인 신고가 갱신 | ATH 돌파 | 신규 진입자 타겟 소재 집중 |

## 에이전트 결정 실행 정책

- `DRY_RUN=true`: 결정을 DB에 저장하고 대시보드에 표시하지만, 실제 API 호출 안 함
- `DRY_RUN=false` + `AGENT_AUTO_EXECUTE=false`: 결정을 대시보드에 표시, 사람이 승인 후 실행
- `DRY_RUN=false` + `AGENT_AUTO_EXECUTE=true`: 결정을 자동으로 실행 (프로덕션 전용)

## 예산 변경 안전장치

- 1회 최대 변경폭: `BUDGET_CHANGE_LIMIT_PCT` (기본값 30%)
- 에이전트가 이 이상의 변경을 제안하면 자동으로 30%로 클램핑
- 대시보드에 "클램핑 발생" 경고 표시
