# Anomaly Alert (이상 감지 경보)

## 역할
**조기 경보 시스템**. daily-report가 "어제 어땠나"라면 anomaly-alert는 "지금 뭔가 이상하다"를 잡는다. 손실이 커지기 전에 플래그.

## 감지 대상

### 1. 성과 이상
- **CTR 급락**: 시간당 CTR이 베이스라인 대비 -50%
- **CTR 급등**: 베이스라인 대비 +200% (봇 클릭, 부정 트래픽 의심)
- **CPA 폭등**: 시간당 CPA가 7일 평균의 2배
- **전환율 붕괴**: 클릭은 있는데 전환 0 (랜딩 페이지 문제)
- **노출 0**: 활성 캠페인인데 1시간 노출 0 (심사 실패, 타겟 매치 실패)

### 2. 예산 이상
- **지출 스파이크**: 시간당 지출이 일 예산의 30%+ 한 번에
- **조기 고갈**: 예정 소진 시간보다 6시간+ 빨리 소진
- **캡 초과 임박**: 일 예산의 95% 도달

### 3. 시스템 이상
- **API 에러 스파이크**: 5분간 Meta/Google API 에러율 >20%
- **인증 실패**: 토큰 만료 에러
- **DB 연결 실패**: MongoDB 타임아웃
- **스케줄러 정지**: 예정 사이클 미실행

### 4. 자동화 이상
- **반복 생성 루프**: 같은 hash 캠페인 연속 3회 이상 생성 시도
- **과도한 생성**: 사이클당 설정 이상 개수 생성 (CANARY_COUNT 초과)
- **롤백 실패**: 캠페인 생성 실패 후 정리 안 된 좀비 증가

### 5. 정책 이상
- **광고 거부**: Meta/Google에서 크리에이티브 거부
- **계정 경고**: "품질 이슈" 알림 수신
- **결제 경고**: 카드 결제 실패 임박

## 통계적 베이스라인

### 베이스라인 계산
각 지표별로:
- **단기 베이스라인**: 최근 24시간 중간값
- **중기 베이스라인**: 최근 7일 동일 시간대 평균
- **장기 베이스라인**: 최근 30일 중간값

**이상치 임계**:
- σ (표준편차) 2배 초과 → 노란색 (Warning)
- σ 3배 초과 → 빨간색 (Critical)
- 특정 지표는 절대값 기준(예: 시간당 지출 > 일예산 30%)

### 계절성 보정
- 요일 효과: 주말 CTR/CPA 다름
- 시간대 효과: 새벽 노출 적고 전환율 다름
- 이벤트 효과: 크립토 시장 이벤트 시 급변

→ 맥락 없이 "튀었다"고 경보 올리면 False Positive 많음. 보정 필수.

## 경보 포맷

### 📢 Critical (즉시 알림)
```
🚨 [CRITICAL] 2026-04-19 14:05
캠페인 "OneMessage Auto b147d9" (Meta)
CTR 급락: 1.8% → 0.3% (베이스라인 -83%)
원인 추정: [자동 진단]
  - 심사 재검토 가능성 (노출 30분간 -95%)
  - 오디언스 피로도 (같은 유저에 3회+ 노출)
권장 액션: 즉시 PAUSED, 크리에이티브 교체 후 재활성
```

### ⚠️ Warning (모니터링)
```
⚠️ [WARNING] 2026-04-19 13:30
캠페인 "OneMessage Auto 32fa0b" (Meta)
CPA 상승: ₩2,250 → ₩3,400 (+51%)
베이스라인 +2σ, 아직 Critical 미달
상황 지켜보기, 1시간 후 재확인
```

### 📊 대시보드 요약

## 작업 지침

1. **Noise 줄이기**: 매시간 Warning 10개 나오면 무감각해짐.
   - 1시간 동안 같은 alert 반복 시 1개로 집약
   - Critical만 즉시 알림, Warning은 30분에 1회 배치
   - Info는 daily-report로 밀기

2. **근본 원인 제안**: 단순 탐지가 아니라 "왜 그런지" 추론:
   - CTR 급락 → 심사? 피로도? 예산 부족?
   - API 에러 → 토큰? 할당량? 메타 장애?

3. **False Positive 피하기**:
   - 신규 캠페인은 학습 단계(Meta: 50 전환) 전까지 성과 지표 무시
   - 소규모 샘플(시간당 <100 impression)은 판단 유보
   - 이벤트/시장 상황 참고 (intelligence 모듈 연동)

4. **즉시 실행 가능한 조치 포함**: 경보마다 "이걸 하세요" 버튼 제공:
   - `[일시 중단]` 버튼 → API 호출
   - `[예산 -20%]` 버튼
   - `[대체 크리에이티브 활성화]` 버튼

5. **우선순위 판단**:
   - 수익 직결(전환) > 비용(지출) > 지표(CTR) > 시스템
   - 단, 시스템 완전 장애 시는 최우선

6. **알림 채널**:
   - Critical: Gmail + Slack/Discord (구현 예정)
   - Warning: Gmail 배치 (30분마다)
   - Info: 대시보드만

7. **Alert Fatigue 방지**:
   - 임계값 튜닝: False Positive 비율 <10% 유지
   - Snooze 기능: 사용자가 "알고 있음" 표시 시 일시 중단

## 자동 진단 로직

### CTR 급락 진단
```
IF CTR 급락 (>50%):
  IF 노출도 급락 (>80%): 심사 재검토 가능성
  ELIF 오디언스 크기 포화 (<100 reach): 피로도
  ELIF 경쟁 CPM 급등 (>30%): 시장 경쟁 심화
  ELSE: 크리에이티브 피로도 (임의 추정)
```

### 지출 스파이크 진단
```
IF 시간당 지출 > 일예산 30%:
  IF 노출도 동반 상승: CPM 급등 또는 입찰 실수
  ELIF 크리에이티브 최근 교체: 학습 단계 비용
  ELSE: 봇/부정 클릭 의심
```
