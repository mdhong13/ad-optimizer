# Campaign Retrospective (캠페인 회고)

## 역할
**패배에서도 배우는 회고**. 캠페인이 끝났을 때(종료/아카이브/사이클 탈락) 단순히 성과 지표를 기록하는 게 아니라, **"왜"를 추론하고 학습을 누적**한다. 이 누적된 학습이 다음 캠페인 설계(audience-analysis, ad-copy)에 피드백되는 선순환 구조.

## 회고 프레임워크: KPT
- **K**eep: 계속 유지할 것 (무엇이 효과적이었나)
- **P**roblem: 문제였던 것 (무엇이 안 됐나)
- **T**ry: 다음에 시도할 것 (학습 적용)

## 회고 대상
- 사이클 탈락 캠페인 (상위 2개 외 하위 18개)
- 수동 중단 캠페인
- 자연 종료 (예산 소진, 기간 만료)
- 이상 종료 (에러, 정책 위반)

## 분석 차원

### 1. 성과 결과
- **Primary Metric 달성**: 목표 CPA/CTR 대비 실제
- **예산 효율**: 예산 대비 전환 수
- **생명주기**: 생성→활성→종료 소요 시간
- **학습 단계 통과 여부**: Meta 50 전환 기준

### 2. 가설 검증
A/B 실험으로 시작했다면:
- 사전 가설(H1)이 맞았나 / 틀렸나 / 불명확한가
- 통계적 유의성 달성됐나 (p < 0.05)
- 예상 효과 크기(MDE)와 실제 차이
- **가설이 틀렸어도 학습** — "공포 앵글이 사랑보다 나쁘다"도 가치 있는 학습

### 3. 성과 귀인 (Attribution)
좋았거나 나빴던 이유 추론:
- **크리에이티브**: 헤드라인/이미지/앵글
- **타겟팅**: 오디언스 정의, Reach 크기
- **입찰**: 입찰 전략, 예산 수준
- **타이밍**: 시장 이벤트, 요일/시간대
- **플랫폼**: 알고리즘 특성, 경쟁 상황
- **외부**: 경쟁사 광고, 시즌 효과

**한 요인으로 단정하지 말고 다중 요인 고려**.

### 4. 기회비용
이 캠페인에 쓴 예산을 다른 곳에 썼다면? 
- 가장 잘한 캠페인이 이 예산을 썼으면 예상 성과
- 실제 성과와의 gap = 기회비용

## 학습 카테고리

회고에서 도출된 학습을 태깅해서 DB에 저장:
- `#persona` — 타겟 정의 관련
- `#angle` — 메시지 각도 (공포/사랑/효율 등)
- `#format` — 광고 포맷 (이미지/영상/캐러셀)
- `#creative` — 특정 크리에이티브 요소 (색상/폰트/이미지 타입)
- `#bidding` — 입찰/예산 전략
- `#timing` — 시간/요일/이벤트
- `#platform` — 플랫폼 특성

→ 다음 `audience-analysis`, `ad-copy`, `ab-test` 실행 시 DB에서 관련 태그 학습 조회해 반영.

## 회고 리포트 포맷

```markdown
# 📝 Campaign Retro — OneMessage Auto 32fa0b (Meta)

## 기본 정보
- **기간**: 2026-04-19 14:01 ~ 2026-04-21 22:00 (2일 8시간)
- **예산 집행**: ₩8,500 / ₩8,000 (106%)
- **결과**: 🏆 사이클 Top 2 (생존)

## 핵심 지표
| 지표 | 실제 | 목표 | 달성 |
|------|------|------|------|
| CPA | ₩1,850 | ₩3,000 | 162% |
| CTR | 2.1% | 1.5% | 140% |
| 전환 | 12 | 8 | 150% |
| 학습 단계 | ✅ 통과 (50 전환 기준 해당 없음, 소규모) |

## 가설 검증
- **H1**: "사랑 앵글이 공포 앵글보다 CTR 20%+ 높을 것"
- **결과**: 사랑 앵글 CTR 2.1% vs 공포 앵글 1.3% → **+61%**
- **통계**: p=0.03 (유의) — 단 샘플 작아 신뢰구간 넓음
- **결론**: ✅ H1 채택. 사랑 앵글이 명확히 우수

## KPT
### Keep (유지)
- 🟢 사랑 앵글 ("가족이 이어받을 수 있게")
- 🟢 파란색 이미지 (4종 중 성과 상위)
- 🟢 "Learn More" CTA

### Problem (문제)
- 🔴 예산 소진 시간이 불균일 (첫날 18시간, 둘째날 14시간에 90%)
- 🔴 아침 시간대 노출 대비 전환 저조

### Try (다음 시도)
- 🟡 아침 시간대 예산 할당 축소 (스케줄 조정)
- 🟡 사랑 앵글 다른 변형 생성 (자녀/배우자/부모 등 구체 대상)
- 🟡 파란 계열 이미지 3장 추가 생성

## 학습 태그
- #angle: 사랑 앵글 (vs 공포) 효과 확인 (+61% CTR)
- #creative: 파란색 이미지 유리 (4종 중 Top 2)
- #timing: 아침 시간대 효율 낮음 (보정 필요)
- #persona: "가족 책임감 있는 30대" 페르소나 강한 반응

## 다음 캠페인에 적용할 것
1. ad-copy 생성 시: 사랑 앵글 우선, 공포 앵글 소수만
2. creative-brief 시: 파란 계열 이미지 요청
3. ab-test 시: "사랑-자녀 vs 사랑-배우자" 같은 세부 앵글 실험
```

## 작업 지침

1. **감정 배제**: 내가 만든 캠페인이라도 냉정하게. "좋았다"보다 "지표가 얼마나 나왔다" 먼저.

2. **다중 요인 고려**: 승패 원인을 단 하나로 단정하지 말 것. 상관관계 ≠ 인과관계.

3. **샘플 크기 조심**: 소규모 샘플(전환 10개 이하) 결과는 강한 결론 내리지 말고 "추정"으로.

4. **외부 요인 체크**: 시장 이벤트(intelligence 모듈), 경쟁사 움직임 등 캠페인 외 요인 참고.

5. **학습 태그 필수**: 태그 없는 회고는 학습 누적 안 됨. 최소 2~3개 태그.

6. **DB 저장 포맷**:
   ```json
   {
     "campaign_id": "...",
     "retro_date": "2026-04-21",
     "kpt": {...},
     "learnings": [
       {"tag": "#angle", "insight": "사랑 앵글 우수", "confidence": 0.8}
     ],
     "next_actions": [...]
   }
   ```

7. **부정적 학습도 기록**: 실패에서 배운 것도 동등하게 가치. "공포 앵글은 소규모 샘플에선 성과 나빴다" 같은 것.

8. **통합 회고 vs 개별 회고**:
   - 개별: 1 캠페인 = 1 회고
   - 통합: 1 사이클 20개 = 1 종합 회고 (패턴 찾기)
   - 둘 다 생성, 통합은 매 사이클마다

9. **선순환 닫기**: 회고 학습 → audience-analysis / ad-copy / ab-test에 반영 여부 추적. 학습이 실제로 적용됐는지 확인.

## OneMessage 프로젝트 맥락
- 예산 작음 → 단일 캠페인 전환 수 적어 통계적 유의성 어려움
- 장기 학습 누적이 핵심 → 개별 회고도 꾸준히
- DRY_RUN 데이터도 회고 가능 (가상 성과라도 의사결정 연습)
- 회고 결과는 `campaign_retros` 컬렉션에 저장
