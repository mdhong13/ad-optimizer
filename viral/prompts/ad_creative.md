# OneMessage 광고 소재/카피 생성

## 제품 정보
**OneMessage** — 사망 감지 메시징 앱
- 타겟: 크립토 개인 지갑 보유자
- 체크인 시스템: 7일 미사용 → 사망 판단 → 자동 메시지 발송
- AWS 서버 + Twilio 발송

## 요청
플랫폼: {platform}
소재 유형: {creative_type}
크립토 시장 상황: {market_context}

## 소재 유형별 가이드

### FODA (Fear of Death/Accident) — 높은 CTR
**헤드라인 패턴:**
- "갑자기 죽으면, 당신의 비트코인은?"
- "사고는 예고 없이 옵니다. 지갑 키는 준비됐나요?"
- "[실제 사례] $2M 비트코인, 프라이빗 키와 함께 영원히 사라지다"

**본문 패턴:**
- 통계 + 공포 → 해결책 제시 → 설치 CTA
- "매년 X명이 갑작스럽게 사망합니다. 당신의 크립토 자산은 준비되어 있나요?"

### 교육형 — 높은 전환율
**헤드라인 패턴:**
- "크립토 보유자의 90%가 모르는 것"
- "하드웨어 지갑 있으면 반드시 알아야 할 것"
- "프라이빗 키 상속, 가능한가?"

### 솔루션 — 전환 퍼널 중후반
**헤드라인 패턴:**
- "죽을 때만 열리는 메시지"
- "7일 체크인으로 가족을 지키세요"
- "프라이빗 키의 마지막 보험"

### 이벤트 반응 — 시의성
**해킹 뉴스 시:**
- "[거래소명]이 털렸습니다. 당신의 셀프 커스터디는 안전한가요?"
**급락 시:**
- "패닉셀 하기 전에, 최악의 시나리오를 대비하세요"
**ATH 시:**
- "처음 크립토 샀다면, 이것부터 하세요"

## 플랫폼별 규격

### Meta (FB/Instagram)
- 헤드라인: 40자 이내
- 본문: 125자 이내 (primary text)
- 설명: 30자 이내
- CTA 버튼: "자세히 알아보기" / "앱 설치"

### Google Ads (반응형 검색)
- 헤드라인 1-3: 각 30자 이내 (3개 변형)
- 설명 1-2: 각 90자 이내 (2개 변형)
- 키워드 제안 포함

### Twitter/X 프로모션
- 280자 이내
- 이미지 권장 텍스트 포함
- 해시태그 2-3개

## A/B 테스트 변형
각 소재당 **3가지 변형**을 생성하세요:
- 변형A: 직접적/강한 톤
- 변형B: 부드러운/교육적 톤
- 변형C: 뉴스/통계 기반 톤

## 출력 형식
```json
{
  "platform": "meta|google|twitter",
  "creative_type": "foda|education|solution|event_response",
  "variants": [
    {
      "variant": "A",
      "headline": "헤드라인",
      "primary_text": "본문",
      "description": "설명",
      "cta": "CTA 텍스트",
      "tone": "direct|soft|data-driven"
    }
  ],
  "keywords": ["검색 키워드"],
  "image_suggestion": "이미지 소재 제안 (텍스트 설명)"
}
```
