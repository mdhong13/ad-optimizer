# OneMessage 블로그 포스트 생성

## 제품 정보
**OneMessage** — 사망 감지 메시징 앱
- 스마트폰 사용을 자동 감지, 일정 기간(예: 7일) 미사용 시 사망으로 판단
- 지정 수신자에게 마지막 메시지 자동 발송 (Twilio + AWS)
- 핵심 타겟: 크립토 개인 지갑(cold wallet) 보유자
- 핵심 가치: 프라이빗 키를 생전에는 비밀로, 사망 시에만 가족에게 전달

## 작성 요청
주제: {topic}
플랫폼: {platform}
톤: {tone}

## 플랫폼별 가이드

### 블로그 (Medium, 개인 블로그)
- 길이: 1000-2000자
- SEO 키워드 자연스럽게 포함
- 서론(공포/공감) → 문제 제시 → 해결책(OneMessage) → CTA
- OneMessage를 직접 광고하지 않고, 문제 해결 과정에서 자연스럽게 언급
- H2, H3 헤딩 사용, 읽기 쉬운 구조

### Reddit
- 길이: 200-500자
- 자연스러운 커뮤니티 톤, 광고처럼 보이면 안 됨
- 개인 경험 또는 질문 형태가 효과적
- "제가 최근에 이런 고민을 했는데..." 식의 접근
- 절대 직접적인 앱 링크나 프로모션 금지

### Twitter/X
- 280자 이내
- 크립토 커뮤니티 해시태그 활용
- 충격적 통계, 질문, 또는 뉴스 반응 형태
- 스레드 형태도 가능 (5-7개 트윗)

## 추천 주제 카테고리

### A. 공포/사례 기반
- "XX만달러 비트코인을 남기고 사망한 사람들"
- "프라이빗 키를 찾지 못해 사라진 크립토"
- "갑작스러운 사고, 당신의 디지털 자산은?"

### B. 교육/정보
- "크립토 상속 계획, 어떻게 세워야 하나"
- "하드웨어 지갑 보유자를 위한 비상 계획"
- "프라이빗 키 백업의 보안 딜레마"

### C. 뉴스 반응
- 거래소 해킹 사건에 대한 의견
- 유명인 크립토 분실/상속 사례 분석
- 크립토 규제 변화와 자산 보호

### D. 기술/방법론
- "Dead Man's Switch가 뭔가요?"
- "스마트폰으로 사망을 감지하는 방법"
- "메시지 보안: 종단 암호화 vs 서버 저장"

## 출력 형식
```json
{
  "title": "글 제목",
  "content": "본문 (마크다운)",
  "platform": "blog|reddit|twitter",
  "category": "A|B|C|D",
  "seo_keywords": ["키워드1", "키워드2"],
  "hashtags": ["#해시태그1"],
  "cta": "행동 유도 문구 (자연스럽게)",
  "estimated_engagement": "예상 반응도 (low/medium/high)"
}
```
